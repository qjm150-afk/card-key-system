# 卡密管理后台使用说明书

## 一、概述

本系统是一个卡密验证管理系统，用于管理用户访问权限。用户在前端输入卡密验证后，可访问对应的飞书文档内容。

**后台地址**：`/admin`

---

## 登录说明

### 默认密码

```
QJM150
```

**⚠️ 重要提示**：部署后请立即修改默认密码！

### 修改密码

在环境变量中设置 `ADMIN_PASSWORD`：

```
ADMIN_PASSWORD=你的新密码
```

- 登录后 **24小时** 内有效
- 超时后需重新登录

| 按钮 | 功能 |
|-----|------|

| **📚 文档中心** | 查看项目文档（开发规范、部署指南等） |
| **退出登录** | 退出管理后台 |

## 三、卡密管理

### 3.1 高级筛选

支持多维度筛选卡密：

| 筛选维度 | 说明 |
|---------|------|
| **搜索框** | 支持按卡密内容或备注内容搜索 |
| **激活状态** | 可筛选"全部"、"有效"、"已激活"、"已停用" |
| **销售状态** | 可筛选"未售出"、"已售出"、"已退款"、"有纠纷" |
| **销售渠道** | 下拉选择数据库中所有不同的销售渠道 |
| **绑定设备** | 筛选绑定设备数量（0-5台） |
| **过期时间** | 可筛选"已过期"、按日期分组、"永久有效" |
| **链接名称** | 下拉选择数据库中所有不同的飞书链接 |
| **创建时间** | 选择日期范围筛选 |

**筛选逻辑：**
- 所有筛选条件为 **AND** 关系（同时满足）
- 每个筛选下拉框的选项会根据当前其他筛选条件**动态更新**
- 当前字段的下拉框保持**完整选项**（排除自身筛选条件）

**示例**：
- 选择"过期时间 = 2026-12-26 到期"后：
  - 过期时间下拉框仍显示所有选项
  - 销售渠道下拉框只显示符合条件的渠道（如"淘宝"）

**筛选操作：**
- 点击「搜索」执行筛选
- 点击「重置」清空所有筛选条件
- 已选条件以标签形式展示，可点击 × 移除单个条件

### 3.2 全选筛选结果

- 筛选后显示"筛选结果: X 条"
- 点击「全选筛选结果」可一键选中所有筛选出的卡密（支持跨页）

### 3.3 批量操作

勾选表格中的卡密后，可执行以下批量操作：

| 按钮 | 功能 | 业务规则 |
|-----|------|---------|
| **批量激活** | 将选中的卡密激活状态改为"有效" | 不影响销售状态 |
| **批量停用** | 将选中的卡密激活状态改为"已停用" | 不影响销售状态 |
| **批量改链** | 批量修改选中卡密的飞书链接 | 显示当前链接分布预览 |
| **导入销售状态** | 通过CSV文件批量导入销售状态 | 已退款/有纠纷自动停用 |
| **批量删除** | 删除选中的卡密及其访问日志 | 不可恢复 |

## 六、批量改链

用于批量修改选中卡密的飞书链接，适用于不同时间段生成的卡密对应不同链接的更新场景。

### 操作流程

1. 筛选出需要修改的卡密（如按飞书链接筛选）
2. 点击「全选筛选结果」
3. 点击「批量改链」按钮
4. 查看当前链接分布预览
5. 输入新的飞书链接和访问密码
6. 点击「确认修改」

### 弹窗信息

| 信息项 | 说明 |
|-------|------|
| **已选择卡密** | 显示选中的卡密数量 |
| **当前链接分布** | 显示选中卡密的链接统计（链接 + 数量） |
| **新飞书链接** | 必填，要替换的新链接 |

## 七、销售状态管理

### 7.1 销售状态类型

| 状态 | 值 | 说明 | 颜色 |
|-----|-----|------|------|
| 未售出 | unsold | 默认状态，尚未销售 | 灰色 |
| 已售出 | sold | 已成功销售 | 绿色 |
| 已退款 | refunded | 用户申请退款 | 红色 |
| 有纠纷 | disputed | 存在交易纠纷 | 橙色 |

### 7.2 导入销售状态

用于批量更新卡密的销售状态，支持根据阿奇索等平台的订单数据进行更新。

#### 操作流程

1. 点击「导入销售状态」按钮
2. 点击「下载模板」获取CSV模板
3. 填写卡密、订单号、销售状态
4. 上传填写好的CSV文件
5. 系统自动匹配更新

#### CSV模板格式

```csv
卡密,订单号,销售状态
CSS-XXXX-XXXX-XXXX,ORDER123456,已售出
CSS-YYYY-YYYY-YYYY,ORDER789012,已退款
CSS-ZZZZ-ZZZZ-ZZZZ,ORDER111111,有纠纷
```

#### 销售状态可选值

- 未售出
- 已售出
- 已退款
- 有纠纷

### 7.3 自动停用规则 ⚠️

**重要业务规则**：导入销售状态时，系统会自动处理激活状态：

| 销售状态 | 自动操作 |
|---------|---------|
| 已售出 | 记录售出时间，不改变激活状态 |
| 已退款 | **自动停用**（激活状态改为"已停用"） |
| 有纠纷 | **自动停用**（激活状态改为"已停用"） |

**设计原因**：已退款或有纠纷的卡密不应继续被使用，系统自动停用以保护商家利益。

## 九、设备绑定机制说明

### 9.1 设备限制规则

- 每个卡密最多可在 **5台设备** 上使用
- 设备ID由浏览器自动生成并存储在 localStorage
- 清除浏览器数据会生成新的设备ID（占用新设备名额）

### 9.2 验证流程

1. 用户输入卡密 → 系统检查卡密状态
2. 检查是否过期 → 检查设备数量
3. 如果是新设备且未达上限 → 绑定新设备，验证成功
4. 如果是已绑定设备 → 直接验证成功
5. 如果设备数量已达上限 → 验证失败，提示"该卡密已在5台设备上使用"

### 9.3 会话持久化

- 验证成功后，会话保存在浏览器 localStorage
- 会话有效期：**30天**
- 30天内再次访问无需重新验证
- 同一设备重复访问不会占用新设备名额

## 十、业务规则汇总

### 10.1 激活状态 vs 销售状态

**激活状态说明**：
- **有效**：status=1，卡密可用，未被使用过
- **已激活**：status=1 且 used_count>0，卡密可用，已被使用过（系统自动判断，不可手动设置）
- **已停用**：status=0，卡密不可用

**销售状态说明**：
- **未售出**：卡密尚未销售
- **已售出**：卡密已销售成功
- **已退款**：卡密已退款
- **有纠纷**：卡密存在交易纠纷

| 激活状态 | 销售状态 | 说明 |
|---------|---------|------|
| 有效 | 未售出 | 可正常使用，尚未销售 |
| 有效 | 已售出 | 可正常使用，已销售成功 |
| 已停用 | 已退款 | 退款后自动停用 |
| 已停用 | 有纠纷 | 纠纷后自动停用 |
| 已停用 | 任意 | 管理员手动停用 |

### 10.2 状态联动规则

| 操作 | 激活状态变化 | 销售状态变化 |
|-----|-------------|-------------|
| 批量激活 | 有效 | 不变 |
| 批量停用 | 已停用 | 不变 |
| 导入"已退款" | **自动停用** | 已退款 |
| 导入"有纠纷" | **自动停用** | 有纠纷 |
| 导入"已售出" | 不变 | 已售出 |

### 10.3 数据导出格式

导出卡密时，默认采用阿奇索平台兼容格式：
- 文件格式：CSV
- 内容格式：卡号,密码（无表头）
- 编码：UTF-8

## 十一、常见问题

### Q1：如何让一个卡密供多人使用？
A：一个卡密最多支持5台设备，如果需要更多用户，请创建多个卡密。

### Q2：用户更换设备后无法验证？
A：设备绑定后不会自动解绑。如需在新设备使用，管理员可以：
- 在数据库中将该卡密的 `devices` 字段清空（重置设备绑定）
- 或者创建新卡密

### Q3：如何让卡密在一定时间后失效？
A：在添加/编辑卡密时设置"过期时间"，过期后卡密自动失效。

### Q4：用户清除浏览器缓存后占用新设备名额？
A：是的，清除缓存会生成新的设备ID。管理员可在数据库中清理 devices 字段解决。

### Q5：如何批量更换飞书链接？
A：使用筛选功能按飞书链接筛选，全选后点击「批量改链」即可。

### Q6：导入销售状态后卡密被停用了？
A：已退款和有纠纷的卡密会自动停用，这是系统保护机制。如需重新启用，请手动修改激活状态。

### Q7：如何查看某个卡密属于哪个订单？
A：在编辑卡密时可查看订单号，或在数据库中查询 order_id 字段。

## 十三、安全保护说明

### 前端保护措施

系统已添加以下前端保护措施：

| 保护措施 | 说明 |
|---------|------|
| **禁用右键菜单** | 阻止右键弹出菜单 |
| **禁用复制/剪切** | 阻止 Ctrl+C/X 快捷键复制 |
| **禁用选择文本** | 阻止鼠标选择文本内容 |
| **禁用拖拽** | 阻止拖拽页面元素 |
| **禁用保存/打印** | 阻止 Ctrl+S/P 快捷键 |
| **禁用查看源码** | 阻止 Ctrl+U 快捷键 |
| **禁用开发者工具** | 阻止 F12、Ctrl+Shift+I/J/C |
| **开发者工具检测** | 检测到开发者工具时显示警告 |
| **iframe sandbox** | 限制 iframe 内的功能 |

### iframe Sandbox 限制

iframe 已添加 sandbox 属性，仅允许必要功能：
- `allow-scripts` - 允许脚本执行（必需）
- `allow-same-origin` - 允许同源访问（必需）
- `allow-forms` - 允许表单提交
- `allow-popups` - 允许弹出窗口
- `allow-popups-to-escape-sandbox` - 允许弹出窗口脱离沙箱

**禁止的功能**：
- `allow-downloads` - 禁止下载
- `allow-top-navigation` - 禁止顶层导航

### ⚠️ 重要提示

**前端保护措施的局限性**：
- 前端保护可通过技术手段绕过
- 如需更强的数据安全，请在飞书端设置：
  - 限制分享权限
  - 设置访问密码
  - 关闭导出/复制权限
  - 设置水印

### 飞书端安全设置建议

1. **分享设置**：
   - 仅允许指定人员访问
   - 设置访问有效期
   - 关闭"允许下载"

2. **权限设置**：
   - 设置为"仅查看"
   - 关闭"复制内容"
   - 关闭"导出"

3. **安全设置**：
   - 开启水印功能
   - 设置访问密码
