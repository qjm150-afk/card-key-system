# 卡密管理后台使用说明书

## 一、概述

本系统是一个卡密验证管理系统，用于管理用户访问权限。用户在前端输入卡密验证后，可访问对应的飞书文档内容。

**后台地址**：`/admin`

---

## 二、统计卡片说明

后台首页顶部显示三个统计卡片：

| 卡片名称 | 含义 |
|---------|------|
| **总卡密数** | 系统中所有卡密的总数量（包含有效和无效状态） |
| **有效卡密** | 当前状态为"有效"的卡密数量（status=1） |
| **已绑定设备** | 所有卡密已绑定的设备总数（每个卡密最多绑定5台设备） |

| **批量删除** | 删除选中的卡密及其访问日志 |

## 七、设备绑定机制说明

### 7.1 设备限制规则

- 每个卡密最多可在 **5台设备** 上使用
- 设备ID由浏览器自动生成并存储在 localStorage
- 清除浏览器数据会生成新的设备ID（占用新设备名额）

### 7.2 验证流程

1. 用户输入卡密 → 系统检查卡密状态
2. 检查是否过期 → 检查设备数量
3. 如果是新设备且未达上限 → 绑定新设备，验证成功
4. 如果是已绑定设备 → 直接验证成功
5. 如果设备数量已达上限 → 验证失败，提示"该卡密已在5台设备上使用"

### 7.3 会话持久化

- 验证成功后，会话保存在浏览器 localStorage
- 会话有效期：**30天**
- 30天内再次访问无需重新验证
- 同一设备重复访问不会占用新设备名额

## 八、常见问题

### Q1：如何让一个卡密供多人使用？
A：一个卡密最多支持5台设备，如果需要更多用户，请创建多个卡密。

### Q2：用户更换设备后无法验证？
A：设备绑定后不会自动解绑。如需在新设备使用，管理员可以：
- 在数据库中将该卡密的 `devices` 字段清空（重置设备绑定）
- 或者创建新卡密

### Q3：如何让卡密在一定时间后失效？
A：在添加/编辑卡密时设置"过期时间"，过期后卡密自动失效。

### Q4：用户清除浏览器缓存后占用新设备名额？
A：是的，清除缓存会生成新的设备ID。管理员可在数据库中清理 devices 字段解决。

### Q5：如何批量更换飞书链接？
A：使用批量操作中的"批量更新链接"功能，或通过数据库直接更新。

## 九、数据库字段参考

### card_keys_table（卡密表）

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | INTEGER | 主键 |
| key_value | TEXT | 卡密值 |
| status | INTEGER | 状态：1=有效，0=无效 |
| devices | TEXT | 已绑定设备ID列表（JSON数组） |
| max_devices | INTEGER | 最大设备数，默认5 |
| expire_at | TIMESTAMP | 过期时间 |
| user_note | TEXT | 备注 |
| feishu_url | TEXT | 飞书链接 |
| feishu_password | TEXT | 飞书访问密码 |
| last_used_at | TIMESTAMP | 最后使用时间 |
| bstudio_create_time | TIMESTAMP | 创建时间 |

### access_logs（访问日志表）

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | INTEGER | 主键 |
| card_key_id | INTEGER | 关联的卡密ID |
| key_value | TEXT | 卡密值 |
| ip_address | TEXT | 访问者IP |
| user_agent | TEXT | 浏览器信息 |
| success | BOOLEAN | 是否成功 |
| error_msg | TEXT | 错误信息 |
| access_time | TIMESTAMP | 访问时间 |
