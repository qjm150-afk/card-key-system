# 卡密管理后台使用说明书

## 一、概述

本系统是一个卡密验证管理系统，用于管理用户访问权限。用户在前端输入卡密验证后，可访问对应的飞书文档内容。

**后台地址**：`/admin`

## 登录说明

### 默认密码

```
QJM150
```

**⚠️ 重要提示**：部署后请立即修改默认密码！

### 修改密码

**方式一：后台在线修改（推荐）**

登录管理后台后，点击头部导航的「修改密码」按钮，即可在线修改密码。

- 需要输入旧密码验证身份
- 新密码长度 4-50 位
- 修改后立即生效

**方式二：环境变量设置**

在环境变量中设置 `ADMIN_PASSWORD`：

```
ADMIN_PASSWORD=你的新密码
```

**注意**：如果使用方式一修改过密码，则优先使用数据库中的密码，环境变量设置将不生效。

### 登录有效期

- 登录后 **24小时** 内有效
- 超时后需重新登录

## 三、卡密管理

### 3.1 高级筛选

支持多维度筛选卡密：

| 筛选维度 | 说明 |
|---------|------|
| **搜索框** | 支持按卡密内容或备注内容搜索 |
| **激活状态** | 可筛选"全部"、"有效"、"已激活"、"已停用" |
| **销售状态** | 可筛选"未售出"、"已售出"、"已退款"、"有纠纷" |
| **销售渠道** | 下拉选择数据库中所有不同的销售渠道 |
| **绑定设备** | 筛选绑定设备数量（0-5台） |
| **过期时间** | 可筛选"已过期"、按日期分组、"永久有效" |
| **链接名称** | 下拉选择数据库中所有不同的飞书链接 |
| **创建时间** | 选择日期范围筛选 |

**筛选逻辑：**
- 所有筛选条件为 **AND** 关系（同时满足）
- 每个筛选下拉框的选项会根据当前其他筛选条件**动态更新**
- 当前字段的下拉框保持**完整选项**（排除自身筛选条件）

**示例**：
- 选择"过期时间 = 2026-12-26 到期"后：
  - 过期时间下拉框仍显示所有选项
  - 销售渠道下拉框只显示符合条件的渠道（如"淘宝"）

**筛选操作：**
- 点击「搜索」执行筛选
- 点击「重置」清空所有筛选条件
- 已选条件以标签形式展示，可点击 × 移除单个条件

### 3.2 全选筛选结果

- 筛选后显示"筛选结果: X 条"
- 点击「全选筛选结果」可一键选中所有筛选出的卡密（支持跨页）

### 3.3 批量操作

## 五、批量生成卡密

| 字段名 | 说明 |
|-------|------|
| **生成数量** | 要生成的卡密数量（1-1000个） |
| **卡密前缀** | 卡密的前缀，默认为 `CSS`（详见下方说明） |
| **有效期（天）** | 从当前时间起的有效天数，不填则永不过期 |
| **飞书链接** | 所有生成的卡密共用此链接 |
| **访问密码** | 飞书文档的访问密码（详见下方说明） |
| **备注** | 所有生成的卡密共用此备注 |

### 5.1 卡密前缀设置说明

| 属性 | 说明 |
|-----|------|
| **长度限制** | 最多 **10位** 字符 |
| **默认值** | `CSS` |
| **允许字符** | 任意英文字母、数字组合（也支持中文） |
| **是否必填** | 否，留空则使用默认值 `CSS` |

**卡密格式规则：**
```
格式: {前缀}-XXXX-XXXX-XXXX
       ↑      ↑
    用户自定义  3组4位16进制字符(0-9, A-F)
```

**示例：**

| 前缀输入 | 生成的卡密示例 | 总长度 |
|---------|---------------|--------|
| `CSS`（默认） | CSS-A1B2-C3D4-E5F6 | 19位 |
| `VIP` | VIP-A1B2-C3D4-E5F6 | 18位 |
| `PRO` | PRO-A1B2-C3D4-E5F6 | 18位 |
| `admin` | admin-A1B2-C3D4-E5F6 | 21位 |
| `TEST2024` | TEST2024-A1B2-C3D4-E5F6 | 24位 |

**⚠️ 注意事项：**
- 前缀区分大小写，建议使用大写字母
- 前缀过长会影响卡密可读性，建议控制在 3-5 位
- 不同前缀可用于区分不同批次或渠道的卡密

### 5.2 访问密码说明

**用途**：飞书文档的访问密码，用于在用户验证成功后显示给用户。

| 场景 | 是否需要填写 |
|-----|-------------|
| 飞书文档设置了访问密码 | ✅ 需要填写 |
| 飞书文档公开访问，无需密码 | ❌ 留空即可 |

**用户体验：**

当填写了访问密码后，用户验证卡密成功时，页面右上角会弹出浮动提示框：
- 显示"访问密码：XXX"
- 提供"复制"按钮，方便用户复制密码
- **不会自动关闭**，需用户手动点击 ✕ 关闭
- 关闭按钮较大，方便点击

**工作流程：**
```
1. 管理员在飞书文档设置了访问密码（如：abc123）
2. 批量生成卡密时，填写这个密码
3. 用户验证卡密成功后，页面弹出密码提示框
4. 用户点击"复制"按钮，粘贴到飞书密码输入框
5. 用户使用完毕后，点击 ✕ 关闭提示框
```

## 七、销售状态管理

### 7.1 销售状态类型

| 状态 | 值 | 说明 | 颜色 |
|-----|-----|------|------|
| 未售出 | unsold | 默认状态，尚未销售 | 灰色 |
| 已售出 | sold | 已成功销售 | 绿色 |
| 已退款 | refunded | 用户申请退款 | 红色 |
| 有纠纷 | disputed | 存在交易纠纷 | 橙色 |

### 7.2 导入销售状态

用于批量更新卡密的销售状态，支持根据阿奇索等平台的订单数据进行更新。

#### 操作流程

1. 点击「导入销售状态」按钮
2. 点击「下载模板」获取CSV模板
3. 填写卡密、订单号、销售状态
4. 上传填写好的CSV文件
5. 系统自动匹配更新

#### CSV模板格式

```csv
卡密,订单号,销售状态
CSS-XXXX-XXXX-XXXX,ORDER123456,已售出
CSS-YYYY-YYYY-YYYY,ORDER789012,已退款
CSS-ZZZZ-ZZZZ-ZZZZ,ORDER111111,有纠纷
```

#### 销售状态可选值

- 未售出
- 已售出
- 已退款
- 有纠纷

### 7.3 自动停用规则 ⚠️

**重要业务规则**：导入销售状态时，系统会自动处理激活状态：

| 销售状态 | 自动操作 |
|---------|---------|
| 已售出 | 记录售出时间，不改变激活状态 |
| 已退款 | **自动停用**（激活状态改为"已停用"） |
| 有纠纷 | **自动停用**（激活状态改为"已停用"） |

**设计原因**：已退款或有纠纷的卡密不应继续被使用，系统自动停用以保护商家利益。

## 九、设备绑定机制说明

### 9.1 设备限制规则

- 每个卡密最多可在 **5台设备** 上使用
- 设备ID由浏览器自动生成并存储在 localStorage
- 清除浏览器数据会生成新的设备ID（占用新设备名额）

### 9.2 验证流程

1. 用户输入卡密 → 系统检查卡密状态
2. 检查是否过期 → 检查设备数量
3. 如果是新设备且未达上限 → 绑定新设备，验证成功
4. 如果是已绑定设备 → 直接验证成功
5. 如果设备数量已达上限 → 验证失败，提示"该卡密已在5台设备上使用"

### 9.3 会话持久化

- 验证成功后，会话保存在浏览器 localStorage
- 会话有效期：**30天**
- 30天内再次访问无需重新验证
- 同一设备重复访问不会占用新设备名额

## 十、业务规则汇总

### 10.1 激活状态 vs 销售状态

**激活状态说明**：
- **有效**：status=1，卡密可用，未被使用过
- **已激活**：status=1 且 used_count>0，卡密可用，已被使用过（系统自动判断，不可手动设置）
- **已停用**：status=0，卡密不可用

**销售状态说明**：
- **未售出**：卡密尚未销售
- **已售出**：卡密已销售成功
- **已退款**：卡密已退款
- **有纠纷**：卡密存在交易纠纷

| 激活状态 | 销售状态 | 说明 |
|---------|---------|------|
| 有效 | 未售出 | 可正常使用，尚未销售 |
| 有效 | 已售出 | 可正常使用，已销售成功 |
| 已停用 | 已退款 | 退款后自动停用 |
| 已停用 | 有纠纷 | 纠纷后自动停用 |
| 已停用 | 任意 | 管理员手动停用 |

### 10.2 状态联动规则

| 操作 | 激活状态变化 | 销售状态变化 |
|-----|-------------|-------------|
| 批量激活 | 有效 | 不变 |
| 批量停用 | 已停用 | 不变 |
| 导入"已退款" | **自动停用** | 已退款 |
| 导入"有纠纷" | **自动停用** | 有纠纷 |
| 导入"已售出" | 不变 | 已售出 |

### 10.3 数据导出格式

导出卡密时，默认采用阿奇索平台兼容格式：
- 文件格式：CSV
- 内容格式：卡号,密码（无表头）
- 编码：UTF-8

## 十一、常见问题

### Q1：如何让一个卡密供多人使用？
A：一个卡密最多支持5台设备，如果需要更多用户，请创建多个卡密。

### Q2：用户更换设备后无法验证？
A：设备绑定后不会自动解绑。如需在新设备使用，管理员可以：
- 在数据库中将该卡密的 `devices` 字段清空（重置设备绑定）
- 或者创建新卡密

### Q3：如何让卡密在一定时间后失效？
A：在添加/编辑卡密时设置"过期时间"，过期后卡密自动失效。

### Q4：用户清除浏览器缓存后占用新设备名额？
A：是的，清除缓存会生成新的设备ID。管理员可在数据库中清理 devices 字段解决。

### Q5：如何批量更换飞书链接？
A：使用筛选功能按飞书链接筛选，全选后点击「批量改链」即可。

### Q6：导入销售状态后卡密被停用了？
A：已退款和有纠纷的卡密会自动停用，这是系统保护机制。如需重新启用，请手动修改激活状态。

### Q7：如何查看某个卡密属于哪个订单？
A：在编辑卡密时可查看订单号，或在数据库中查询 order_id 字段。

### 最大设备数字段支持 - 详细修改清单

**后端 `src/main.py`：**
- `CardKeyCreate` 模型：添加 `max_devices: Optional[int] = 5`
- `CardKeyUpdate` 模型：添加 `max_devices: Optional[int] = None`
- `BatchGenerateRequest` 模型：添加 `max_devices: int = 5`
- `create_card_key` 函数：写入 `max_devices` 字段
- `batch_generate_cards` 函数：写入 `max_devices` 字段
- `update_card_key` 函数：处理 `max_devices` 更新

**前端 `src/static/admin.html`：**
- 编辑/添加表单：添加最大设备数输入框
- 批量生成表单：添加最大设备数输入框
- `editCard` 函数：加载 `max_devices` 数据
- `saveCard` 函数：发送 `max_devices` 数据
- `generateCards` 函数：发送 `max_devices` 数据

**当前替代方案**：可通过导入CSV功能批量修改最大设备数，导入时填写「最大设备数」列即可。

### 密码修改后Token失效 - 详细修改清单

**问题说明**：当前修改密码后，其他已登录用户的 Token 仍可继续使用（24小时有效期），存在安全隐患。

**修改方案**：

**后端 `src/main.py`：**
- `set_admin_password` 函数：成功保存密码后调用 `VALID_TOKENS.clear()` 清除所有 Token
- 返回结果中提示「密码已修改，所有登录已失效，请重新登录」

**前端 `src/static/admin.html`：**
- `handleChangePassword` 函数：修改成功后调用 `handleLogout()` 或跳转到登录页
