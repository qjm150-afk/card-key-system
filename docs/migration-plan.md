# 卡密验证系统迁移方案

## 一、现状分析

### 1.1 当前系统架构
```
┌─────────────────────────────────────────────────────────┐
│                   当前架构                               │
│                                                         │
│   用户 → FastAPI 服务 → Supabase PostgreSQL            │
│                      ↓                                  │
│               飞书多维表格嵌入                           │
└─────────────────────────────────────────────────────────┘
```

### 1.2 当前成本分析

| 成本项 | 费用 | 说明 |
|--------|------|------|
| **Coze 平台托管** | 按用量计费 | 预估 46 万积分/月 ≈ ¥460/月 |
| **Supabase 数据库** | 免费额度 / 超额付费 | 500MB 免费，超出收费 |
| **服务器维护** | 时间成本 | 需定期维护更新 |

### 1.3 当前系统功能

| 功能模块 | 状态 | 依赖 |
|----------|------|------|
| 卡密验证 | ✅ 核心功能 | 服务器 + 数据库 |
| 设备限制 | ✅ 最多 5 台设备 | 数据库 |
| 过期判断 | ✅ 三种过期模式 | 数据库 |
| 管理后台 | ✅ 完整功能 | 服务器 + 数据库 |
| 飞书嵌入 | ✅ CSS 裁剪 | 前端 |
| 预览图片 | ✅ 对象存储 | 存储 |

## 二、迁移方案对比

### 方案 A：飞书协作者方案（推荐 - 最低成本）

**核心思路**：完全放弃 Web 系统，使用飞书原生能力

```
┌─────────────────────────────────────────────────────────┐
│                飞书协作者架构                            │
│                                                         │
│   用户 → 飞书文档 → 申请权限（填写卡密）                 │
│                    ↓                                    │
│              飞书机器人自动审批                          │
│                    ↓                                    │
│            飞书多维表格（卡密存储）                       │
└─────────────────────────────────────────────────────────┘
```

**成本分析**：
| 项目 | 费用 | 说明 |
|------|------|------|
| 飞书机器人 | **¥0** | 企业版免费 |
| 飞书多维表格 | **¥0** | 免费额度充足 |
| 服务器 | **¥0** | 无需服务器 |
| **总成本** | **¥0/月** | 完全免费 |

**功能对比**：
| 功能 | 原 Web 系统 | 飞书协作者方案 | 影响 |
|------|------------|----------------|------|
| 卡密验证 | ✅ | ✅ | 无影响 |
| 过期判断 | ✅ | ✅ | 无影响 |
| 设备限制 | ✅ 最多 5 台 | ❌ 改为用户绑定 | ⚠️ 有影响 |
| 管理后台 | ✅ 自建 | ✅ 飞书多维表格 | 不同体验 |
| 预览图片 | ✅ 自建 | ⚠️ 需重新设计 | 需调整 |

**适合场景**：
- ✅ 预算有限，追求零成本
- ✅ 用户接受使用飞书账号
- ✅ 可以接受「一卡一用户」而非「一卡多设备」

### 方案 B：迁移到免费 Serverless 平台

**核心思路**：保留 Web 系统，迁移到免费平台

```
┌─────────────────────────────────────────────────────────┐
│              Serverless 架构                            │
│                                                         │
│   用户 → Cloudflare Workers/Vercel → Supabase          │
│              ↓                                          │
│        飞书多维表格嵌入                                  │
└─────────────────────────────────────────────────────────┘
```

**平台选择**：

| 平台 | 免费额度 | 适合场景 |
|------|----------|----------|
| **Vercel** | 10万请求/月 | 静态站点 + Serverless Functions |
| **Cloudflare Workers** | 10万请求/天 | 全球 CDN + 边缘计算 |
| **Railway** | $5/月额度 | 小型应用部署 |
| **Render** | 750小时/月 | Web Service |

**成本分析**（以 Cloudflare Workers 为例）：
| 项目 | 费用 | 说明 |
|------|------|------|
| Cloudflare Workers | **¥0** | 10万请求/天免费 |
| Supabase | **¥0** | 500MB 免费 |
| 域名 | ¥50-100/年 | 可选，也可用免费域名 |
| **总成本** | **¥0-10/月** | 接近免费 |

**适合场景**：
- ✅ 需要保留设备限制功能
- ✅ 需要自建管理后台
- ✅ 访问量在免费额度内

---

### 方案 C：混合方案（渐进式迁移）

**核心思路**：双轨运行，逐步迁移

```
┌─────────────────────────────────────────────────────────┐
│                 混合架构                                │
│                                                         │
│   新用户 → 飞书协作者方案（零成本）                       │
│   老用户 → 原 Web 系统（逐步停用）                        │
└─────────────────────────────────────────────────────────┘
```

**迁移步骤**：
1. 新建飞书协作者系统
2. 新卡密全部走飞书方案
3. 老卡密到期后自动迁移
4. 最终关闭 Web 系统

## 三、推荐方案：飞书协作者方案

### 3.1 为什么推荐这个方案？

| 维度 | 分析 |
|------|------|
| **成本** | 完全免费，零运营成本 |
| **维护** | 无服务器、无数据库维护 |
| **安全** | 飞书企业级安全 |
| **合规** | 不收集个人信息 |
| **体验** | 用户直接访问文档，更自然 |

### 3.2 迁移实施计划

#### 第一阶段：准备工作（1-2 天）

```
Step 1: 创建飞书多维表格
    ├── 卡密表（存储卡密信息）
    ├── 验证日志表（记录验证历史）
    └── 导入现有卡密数据

Step 2: 创建飞书应用
    ├── 企业自建应用
    ├── 配置权限（文档、多维表格、消息）
    └── 配置事件订阅

Step 3: 部署机器人
    ├── 选择飞书云函数（推荐）
    └── 或部署到免费 Serverless 平台
```

#### 第二阶段：数据迁移（半天）

使用本地卡密生成工具导出 CSV，导入飞书多维表格：

```bash
# 导出现有卡密
访问：http://localhost:5000/admin → 导出 → CSV

# 导入飞书多维表格
打开飞书多维表格 → 导入 → 选择 CSV 文件
```

#### 第三阶段：测试验证（半天）

```
1. 创建测试卡密
2. 使用测试账号访问文档
3. 申请权限并填写卡密
4. 验证自动审批是否正常
5. 测试过期卡密、已用卡密等场景
```

#### 第四阶段：切换上线（半天）

```
1. 通知用户新的访问方式
2. 更新分享链接
3. 监控运行状态
4. 原 Web 系统可以下线或保留作为备份
```

### 3.3 功能变更说明

| 原功能 | 新方案 | 说明 |
|--------|--------|------|
| **设备限制**（5台） | 用户绑定 | 改为「一卡一用户」，更简单 |
| **管理后台** | 飞书多维表格 | 直接在表格中管理 |
| **预览图片** | 文档封面 | 使用飞书文档自带封面 |
| **访问日志** | 验证日志表 | 记录在多维表格中 |

## 四、本地工具配合使用

### 4.1 工具定位

本地卡密生成工具（`tools/local-card-generator/index.html`）配合飞书协作者方案：

```
┌─────────────────────────────────────────────────────────┐
│               本地工具 + 飞书方案                        │
│                                                         │
│   1. 本地工具：生成卡密 → 导出 CSV                       │
│   2. 飞书多维表格：导入 CSV → 卡密生效                   │
│   3. 飞书机器人：自动审批用户访问                        │
└─────────────────────────────────────────────────────────┘
```

### 4.2 使用流程

```
Step 1: 在本地工具中创建卡种
    └── 只需输入名称，如"VIP资料库"

Step 2: 批量生成卡密
    ├── 选择卡种
    ├── 设置飞书链接
    ├── 设置过期时间
    └── 生成 → 自动下载 CSV

Step 3: 导入飞书多维表格
    ├── 打开飞书多维表格
    ├── 导入 → 选择 CSV
    └── 卡密立即可用
```

## 五、决策建议

### 5.1 如果你的访问量...

| 访问量 | 推荐方案 | 理由 |
|--------|----------|------|
| **< 100 次/天** | 飞书协作者方案 | 零成本，体验好 |
| **100-1000 次/天** | 飞书协作者 或 Serverless | 都可，看功能需求 |
| **> 1000 次/天** | Serverless + Supabase | 需要更稳定的服务 |

### 5.2 如果你需要设备限制功能...

| 需求 | 推荐方案 | 说明 |
|------|----------|------|
| **必须保留** | 方案 B（Serverless） | 飞书方案不支持设备限制 |
| **可以改为用户绑定** | 方案 A（飞书协作者） | 一卡一用户，更简单 |
