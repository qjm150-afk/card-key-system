## 第一步：创建飞书应用

### 1.1 创建应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 点击「创建企业自建应用」
3. 填写应用信息：
   - 应用名称：`卡密验证助手`
   - 应用描述：`自动审批文档权限申请，验证卡密后添加协作者`
   - 应用图标：自定义上传

### 1.2 配置权限

在「权限管理」页面，添加以下权限：

| 权限名称 | 权限说明 | 用途 |
|----------|----------|------|
| `drive:permission:read` | 查看云文档权限 | 查询文档协作者 |
| `drive:permission:write` | 编辑云文档权限 | 添加/移除协作者 |
| `bitable:record:read` | 查看、评论、导出多维表格 | 查询卡密信息 |
| `bitable:record:write` | 编辑多维表格 | 更新卡密状态、记录日志 |
| `im:message:send` | 获取与发送单聊、群组消息 | 发送验证结果通知 |
| `im:message` | 获取与发送单聊、群组消息 | 接收消息（可选） |
| `contact:user.base:readonly` | 获取用户基本信息 | 获取用户信息（可选） |

### 1.3 配置事件订阅

在「事件订阅」页面：

1. 启用事件订阅
2. 配置请求网址（稍后填写，先部署服务）
3. 订阅事件：
   - `drive.permission.apply_event_v1` - 权限申请事件

### 1.4 发布应用

1. 在「版本管理与发布」页面
2. 创建版本并提交审核
3. 审核通过后发布应用

## 第二步：创建多维表格

### 2.1 创建多维表格

1. 打开飞书多维表格
2. 新建空白多维表格
3. 命名为「卡密管理系统」

### 2.2 创建数据表

按照 [02-database-design.md](./02-database-design.md) 的设计：

#### 卡密表 (card_keys)

创建字段：

| 字段名 | 字段类型 |
|--------|----------|
| 卡密 | 文本 |
| 状态 | 单选：有效/已停用/已过期 |
| 文档链接 | 超链接 |
| 过期类型 | 单选：永久/固定日期/激活后N天 |
| 过期时间 | 日期 |
| 有效天数 | 数字 |
| 激活时间 | 日期 |
| 绑定用户 | 成员 |
| 使用次数 | 数字 |
| 最大使用次数 | 数字 |
| 销售状态 | 单选：未售出/已售出/已退款/有纠纷 |
| 创建时间 | 创建时间 |

#### 验证日志表 (access_logs)

创建字段：

| 字段名 | 字段类型 |
|--------|----------|
| 卡密 | 关联字段（关联卡密表）|
| 申请用户 | 成员 |
| 申请理由 | 文本 |
| 验证结果 | 单选：通过/拒绝 |
| 拒绝原因 | 文本 |
| 申请时间 | 创建时间 |
| 文档Token | 文本 |

### 2.3 获取配置信息

从多维表格 URL 中获取 `app_token`：

## 第三步：部署机器人服务

### 方案一：飞书云函数（推荐）

飞书云函数提供免费的 Serverless 部署方案，适合中小规模使用。

#### 3.1 创建云函数

1. 在飞书开放平台，进入「云函数」
2. 创建新函数：
   - 函数名称：`card-key-validator`
   - 运行环境：Node.js 18
   - 内存：256MB
   - 超时时间：30秒

#### 3.2 上传代码

将 `src/` 目录下的代码打包上传：

```bash
# 安装依赖
pnpm install

# 打包
zip -r function.zip src/ node_modules/ package.json

# 上传到飞书云函数控制台
```

#### 3.3 配置环境变量

在云函数配置中添加：

```
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxx
BITABLE_APP_TOKEN=xxxxxxxxxxxxxxxxxx
```

#### 3.4 配置触发器

创建 HTTP 触发器：
- 路径：`/webhook`
- 方法：POST

#### 3.5 配置事件订阅

复制云函数的 URL，填入飞书应用的事件订阅配置。

---

### 方案二：自建服务器

适合需要更多控制权的场景。

#### 3.1 准备服务器

推荐配置：
- CPU：1核
- 内存：1GB
- 带宽：1Mbps

#### 3.2 安装环境

```bash
# 安装 Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 pnpm
npm install -g pnpm
```

#### 3.3 部署代码

```bash
# 克隆代码
git clone <your-repo-url>
cd feishu-card-key-bot

# 安装依赖
pnpm install

# 创建环境变量文件
cp .env.example .env
# 编辑 .env 文件，填入配置
```

#### 3.4 使用 PM2 运行

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start src/index.js --name card-key-bot

# 设置开机自启
pm2 startup
pm2 save
```

#### 3.5 配置 Nginx（可选）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /webhook {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 第四步：配置文档权限

### 4.1 设置文档权限

对于需要保护的飞书文档/多维表格：

1. 打开文档，点击右上角「分享」
2. 在权限设置中：
   - 关闭「获取链接可查看」
   - 关闭「组织内可查看」
   - 仅添加管理员为协作者
3. 用户尝试访问时，需要申请权限

### 4.2 添加机器人

将机器人添加为文档协作者：
1. 在文档分享设置中
2. 搜索机器人应用名称
3. 添加为「可管理」权限

### 4.3 验证流程

用户访问流程：

```
1. 用户访问文档链接
         ↓
2. 提示需要申请权限
         ↓
3. 用户填写申请理由（包含卡密）
         ↓
4. 机器人接收事件
         ↓
5. 验证卡密
         ↓
6. 通过：添加用户为协作者
   拒绝：发送拒绝通知
```

## 常见问题

### Q: 用户申请权限后没有收到回复？

A: 检查以下几点：
1. 机器人应用是否已发布
2. 事件订阅是否正确配置
3. Webhook URL 是否可访问
4. 飞书云函数/服务器日志是否有错误

### Q: 卡密验证失败但提示"卡密不存在"？

A: 检查：
1. 多维表格中卡密格式是否正确
2. 卡密解析正则是否匹配
3. 申请理由格式是否正确（如【卡密】XXX）

### Q: 添加协作者失败？

A: 检查：
1. 机器人是否有文档的管理权限
2. 文档 token 和类型是否正确
3. 用户 ID 是否有效

### Q: 如何处理大量卡密？

A: 建议使用批量导入功能：
1. 准备 CSV 文件
2. 使用多维表格的导入功能
3. 或编写脚本调用 API 批量创建

## 扩展功能

### 1. 卡密批量生成工具

创建脚本批量生成卡密并导入多维表格：

```javascript
// scripts/generate-keys.js
const { generateCardKeys } = require('../src/utils/cardKey');
const { batchCreateCardKeys } = require('../src/services/bitable');

async function main() {
  const cardKeys = generateCardKeys(100, 'VIP');
  const records = cardKeys.map(key => ({
    卡密: key,
    状态: '有效',
    文档链接: 'https://xxx.feishu.cn/xxx',
    过期类型: '激活后N天',
    有效天数: 30
  }));
  
  await batchCreateCardKeys(client, records);
  console.log(`成功创建 ${records.length} 个卡密`);
}

main();
```
