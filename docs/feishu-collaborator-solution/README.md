# 飞书协作者自动审批方案

## 概述

本方案使用飞书文档的协作者模式，实现卡密验证和权限自动审批。用户点击飞书链接后，在申请权限时填写卡密，机器人自动验证并通过审批。

## 核心优势

- **完全免费**：无服务器、无数据库费用，替代扣子托管（预计节省 46 万积分/月）
- **用户体验自然**：点击链接 → 申请权限（填写卡密） → 自动通过
- **无需加群**：用户不需要加入飞书群
- **无需人工审核**：机器人自动处理
- **数据安全**：不收集 IP、UA 等个人信息，符合《个人信息保护法》

## 方案对比

### 与原 Web 系统对比

| 维度 | 原 Web 系统 | 飞书协作者方案 |
|------|------------|----------------|
| **卡密验证** | ✅ 自建验证接口 | ✅ 机器人验证 |
| **过期判断** | ✅ 数据库查询 | ✅ 多维表格查询 |
| **设备限制** | ✅ 按设备 ID 限制 | ❌ 改为飞书用户绑定 |
| **用户管理** | 自建用户系统 | 飞书协作者管理 |
| **数据存储** | PostgreSQL (Supabase) | 飞书多维表格 |
| **服务器成本** | ¥50-200/月 | **¥0** |
| **维护成本** | 需维护服务、数据库 | 仅维护机器人代码 |

### 与扣子托管方案对比

| 维度 | 扣子托管 | 飞书协作者方案 |
|------|----------|----------------|
| **费用** | ~46 万积分/月 ≈ ¥460/月 | **¥0** |
| **功能** | 完整的对话机器人 | 权限自动审批 |
| **用户体验** | 需添加机器人为好友 | 直接访问文档 |
| **灵活性** | 高 | 中（专注权限管理）|

### 核心决策

选择飞书协作者方案的原因：

1. **成本优势**：完全免费，零运营成本
2. **原生体验**：用户直接访问文档，无需额外操作
3. **安全合规**：不收集个人信息
4. **维护简单**：无服务器、无数据库维护

## 快速开始

### 前置条件

- 飞书企业管理员权限
- 飞书开放平台开发者权限
- 可部署服务的环境（飞书云函数或自建服务器）

### 实施步骤（预计 2 小时）

```
Step 1: 创建多维表格（30分钟）
    └─ 按设计文档创建卡密表和日志表
    
Step 2: 创建飞书应用（30分钟）
    └─ 配置权限、事件订阅
    
Step 3: 部署机器人代码（30分钟）
    └─ 选择飞书云函数或自建服务器
    
Step 4: 导入卡密数据（15分钟）
    └─ 从原系统导出并导入
    
Step 5: 测试验证流程（15分钟）
    └─ 端到端测试
```

### 第一步：创建多维表格

参考 [02-database-design.md](./02-database-design.md)，创建以下数据表：

1. **卡密表** - 存储所有卡密信息
2. **验证日志表** - 记录每次验证结果

### 第二步：创建飞书应用

参考 [04-deployment.md](./04-deployment.md)，完成：

1. 创建企业自建应用
2. 配置权限（文档权限、多维表格权限、消息权限）
3. 配置事件订阅（权限申请事件）

### 第三步：部署机器人

参考 [03-bot-code.md](./03-bot-code.md)，选择部署方式：

- **飞书云函数**（推荐）：免费、免维护
- **自建服务器**：更多控制权

### 第四步：数据迁移

从原系统导出卡密数据，导入到飞书多维表格：

| 原字段 | 新字段 |
|--------|--------|
| key_value | 卡密 |
| status | 状态 |
| feishu_url | 文档链接 |
| expire_at | 过期时间 |

### 第五步：测试验证

1. 创建测试卡密
2. 使用测试账号访问文档
3. 申请权限并填写卡密
4. 验证是否自动通过

## 常见问题

### Q: 用户必须使用飞书账号吗？

A: 是的，这是飞书协作者方案的限制。如果用户没有飞书账号，需要先注册。

### Q: 一个卡密可以让多个用户使用吗？

A: 默认是「一卡一用户」。如果需要「一卡多用户」，可以修改代码逻辑，去掉绑定用户检查。

### Q: 如何处理过期卡密？

A: 支持三种过期方式：
- 永久有效
- 固定日期过期
- 激活后 N 天过期

### Q: 用户离职后权限如何收回？

A: 在飞书管理后台可以批量移除协作者，也可以通过机器人 API 自动清理。

### Q: 原系统的设备限制怎么办？

A: 飞书协作者方案无法限制设备数量，改为用户维度限制。如果需要设备限制，建议保留原 Web 系统方案。
