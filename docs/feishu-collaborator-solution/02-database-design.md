# 多维表格设计

## 表结构概览

```
┌─────────────────────────────────────────────────────────┐
│                   飞书多维表格                            │
│                                                         │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
```

## 表1：卡密表 (card_keys)

### 字段定义

| 字段名 | 字段类型 | 是否必填 | 说明 |
|--------|----------|----------|------|
| 卡密 | 文本 | ✅ | 卡密值（唯一），如ABC-1234-5678-XYZ |
| 状态 | 单选 | ✅ | 有效/已停用/已过期，默认"有效" |
| 文档链接 | 超链接 | ✅ | 对应的飞书文档/多维表格链接 |
| 过期类型 | 单选 | ✅ | 永久/固定日期/激活后N天 |
| 过期时间 | 日期 | ❌ | 固定过期时间（过期类型=固定日期时必填） |
| 有效天数 | 数字 | ❌ | 激活后有效天数（过期类型=激活后N天时必填） |
| 激活时间 | 日期 | ❌ | 首次验证时间（自动记录） |
| 绑定用户 | 成员 | ❌ | 已验证的飞书用户 |
| 使用次数 | 数字 | ❌ | 累计使用次数，默认0 |
| 最大使用次数 | 数字 | ❌ | 最大允许使用次数，空表示不限 |
| 销售状态 | 单选 | ❌ | 未售出/已售出/已退款/有纠纷，默认"未售出" |
| 创建时间 | 创建时间 | ✅ | 自动记录 |

### 单选字段选项

**状态字段**：
- 有效
- 已停用
- 已过期

**过期类型字段**：
- 永久
- 固定日期
- 激活后N天

**销售状态字段**：
- 未售出
- 已售出
- 已退款
- 有纠纷

### 示例数据

| 卡密 | 状态 | 文档链接 | 过期类型 | 过期时间 | 有效天数 | 激活时间 | 绑定用户 |
|------|------|----------|----------|----------|----------|----------|----------|
| ABC-1234-0001 | 有效 | https://xxx.feishu.cn/xxx | 永久 | - | - | - | - |
| ABC-1234-0002 | 有效 | https://xxx.feishu.cn/xxx | 固定日期 | 2026-12-31 | - | 2026-04-01 | 张三 |
| ABC-1234-0003 | 有效 | https://xxx.feishu.cn/xxx | 激活后N天 | - | 30 | - | - |
| ABC-1234-0004 | 已停用 | https://xxx.feishu.cn/xxx | 永久 | - | - | - | - |
| ABC-1234-0005 | 已过期 | https://xxx.feishu.cn/xxx | 激活后N天 | - | 7 | 2026-03-20 | 李四 |

## 表2：验证日志表 (access_logs)

### 字段定义

| 字段名 | 字段类型 | 是否必填 | 说明 |
|--------|----------|----------|------|
| 卡密 | 关联字段 | ✅ | 关联卡密表 |
| 申请用户 | 成员 | ✅ | 申请权限的飞书用户 |
| 申请理由 | 文本 | ❌ | 用户填写的申请理由（包含卡密） |
| 验证结果 | 单选 | ✅ | 通过/拒绝 |
| 拒绝原因 | 文本 | ❌ | 拒绝原因 |
| 申请时间 | 创建时间 | ✅ | 自动记录 |
| 文档Token | 文本 | ❌ | 申请的文档Token（用于调试） |

### 单选字段选项

**验证结果字段**：
- 通过
- 拒绝

### 示例数据

| 卡密 | 申请用户 | 申请理由 | 验证结果 | 拒绝原因 | 申请时间 |
|------|----------|----------|----------|----------|----------|
| ABC-1234-0002 | 张三 | 【卡密】ABC-1234-0002 | 通过 | - | 2026-04-01 10:30 |
| - | 王五 | 【卡密】ABC-1234-9999 | 拒绝 | 卡密不存在 | 2026-04-01 11:00 |
| ABC-1234-0003 | 李四 | 卡密：ABC-1234-0003 | 通过 | - | 2026-04-01 14:20 |

## 表3：内容配置表 (content_config)（可选）

### 用途

当一个卡密表对应多个文档时，可以在此表管理文档链接。

### 字段定义

| 字段名 | 字段类型 | 是否必填 | 说明 |
|--------|----------|----------|------|
| 内容名称 | 文本 | ✅ | 链接名称 |
| 文档链接 | 超链接 | ✅ | 飞书文档/多维表格链接 |
| 文档Token | 文本 | ✅ | 文档的唯一标识（从URL中提取） |
| 文档类型 | 单选 | ✅ | 文档/多维表格/文件夹 |
| 默认权限 | 单选 | ✅ | 可查看/可编辑 |
| 是否启用 | 复选框 | ✅ | 默认勾选 |

## 视图设计

### 卡密表视图

#### 1. 全部卡密（默认视图）

- 显示所有卡密
- 按创建时间倒序排列

#### 2. 有效卡密

- 筛选条件：状态 = "有效"
- 用于日常管理

#### 3. 已激活卡密

- 筛选条件：绑定用户 不为空
- 用于查看使用情况

#### 4. 已过期卡密

- 筛选条件：状态 = "已过期" 或（过期类型 = "固定日期" 且 过期时间 < 今天）
- 用于清理过期卡密

### 验证日志表视图

#### 1. 全部日志（默认视图）

- 显示所有日志
- 按申请时间倒序排列

#### 2. 成功记录

- 筛选条件：验证结果 = "通过"

#### 3. 失败记录

- 筛选条件：验证结果 = "拒绝"

## 自动化规则

### 规则1：过期状态自动更新

```
触发条件：每天凌晨0点执行
动作：
  - 查询所有"状态=有效"的卡密
  - 遍历每条卡密：
    - 如果 过期类型 = "固定日期" 且 过期时间 < 今天：
      - 更新状态为"已过期"
    - 如果 过期类型 = "激活后N天" 且 激活时间 + 有效天数 < 今天：
      - 更新状态为"已过期"
```

### 规则2：过期通知（可选）

```
触发条件：卡密状态变更为"已过期"
动作：
  - 发送消息通知绑定用户（如果有）
```

## 数据迁移

### 从原系统迁移卡密数据

原系统的 `card_keys_table` 字段映射：

| 原字段 | 新字段 | 说明 |
|--------|--------|------|
| key_value | 卡密 | 直接迁移 |
| status | 状态 | 1→有效，0→已停用 |
| feishu_url | 文档链接 | 直接迁移 |
| expire_at | 过期时间 | 迁移后设置过期类型为"固定日期" |
| expire_after_days | 有效天数 | 迁移后设置过期类型为"激活后N天" |
| activated_at | 激活时间 | 直接迁移 |
| devices | - | 飞书方案不支持设备限制 |
| sale_status | 销售状态 | 直接迁移 |

### 迁移步骤

1. 导出原系统卡密数据为 CSV
2. 创建飞书多维表格
3. 导入 CSV 到卡密表
4. 调整字段格式（日期、单选等）
5. 验证数据完整性
