## 四、飞书API集成 - 链接泄露检测 🔵

> **需求来源**：用户反馈
> **创建时间**：2026-03-16
> **优先级**：中高
> **状态**：⏳ 规划中

### 4.1 功能概述

通过集成飞书开放平台API，获取多维表格/云文档的访问统计数据，实现以下目标：

1. **泄露检测**：判断飞书链接是否被非法传播
2. **访问分析**：了解用户访问模式，优化运营策略
3. **安全预警**：自动检测异常访问行为

### 4.2 核心功能设计

#### 4.2.1 数据采集

| 数据项 | API来源 | 用途 |
|-------|---------|------|
| 每日访问人数 | 飞书统计API | 趋势分析 |
| 访问来源分布 | 飞书统计API | 内嵌 vs 外部访问判断 |
| 访问设备类型 | 飞书统计API | PC/移动端分布 |
| 访问地域分布 | 飞书统计API | 地域异常检测 |
| 分享/导出次数 | 飞书统计API | 泄露风险评估 |

#### 4.2.2 泄露检测逻辑

```
泄露风险评分 = f(外部访问比例, 异常访问时段, 地域分散度, 分享次数)

风险等级：
- 低风险 (0-30分)：外部访问比例 < 10%，无异常行为
- 中风险 (31-70分)：外部访问比例 10-30%，或存在少量异常
- 高风险 (71-100分)：外部访问比例 > 30%，或明显异常行为

预警触发条件：
1. 外部访问比例突增（较前日增长 > 50%）
2. 单日外部访问人数超过卡密有效数量
3. 非预期时段（如深夜）大量访问
4. 新地域突然出现大量访问
```

#### 4.3.3 数据库设计

**新增表：feishu_link_stats（飞书链接统计）**

```sql
CREATE TABLE feishu_link_stats (
    id SERIAL PRIMARY KEY,
    feishu_url VARCHAR(500) NOT NULL,           -- 飞书链接
    link_name VARCHAR(200),                      -- 链接名称
    stat_date DATE NOT NULL,                     -- 统计日期
    
    -- 访问统计
    total_visitors INT DEFAULT 0,               -- 总访问人数
    embed_visitors INT DEFAULT 0,               -- 内嵌访问人数
    external_visitors INT DEFAULT 0,            -- 外部访问人数
    
    -- 设备分布
    pc_visitors INT DEFAULT 0,                  -- PC端访问
    mobile_visitors INT DEFAULT 0,              -- 移动端访问
    
    -- 其他指标
    share_count INT DEFAULT 0,                  -- 分享次数
    export_count INT DEFAULT 0,                 -- 导出次数
    
    -- 风险评估
    leak_risk_score INT DEFAULT 0,              -- 泄露风险评分(0-100)
    risk_level VARCHAR(20),                     -- 风险等级(low/medium/high)
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(feishu_url, stat_date)               -- 每个链接每天一条记录
);
```

**新增表：feishu_api_config（飞书API配置）**

```sql
CREATE TABLE feishu_api_config (
    id SERIAL PRIMARY KEY,
    app_id VARCHAR(100) NOT NULL,               -- 飞书应用ID
    app_secret VARCHAR(200) NOT NULL,           -- 飞书应用密钥（加密存储）
    tenant_token VARCHAR(500),                  -- 租户访问令牌
    token_expire_at TIMESTAMP,                  -- 令牌过期时间
    
    -- 配置
    sync_enabled BOOLEAN DEFAULT true,          -- 是否启用同步
    sync_interval INT DEFAULT 86400,            -- 同步间隔(秒)，默认每天
    alert_threshold INT DEFAULT 30,             -- 预警阈值(外部访问比例%)
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

**新增表：leak_alerts（泄露预警记录）**

```sql
CREATE TABLE leak_alerts (
    id SERIAL PRIMARY KEY,
    feishu_url VARCHAR(500) NOT NULL,
    link_name VARCHAR(200),
    alert_type VARCHAR(50),                     -- 预警类型
    alert_level VARCHAR(20),                    -- 预警级别
    alert_message TEXT,                         -- 预警信息
    trigger_data JSONB,                         -- 触发数据
    
    status VARCHAR(20) DEFAULT 'pending',       -- pending/handled/ignored
    handled_at TIMESTAMP,
    handled_by VARCHAR(100),
    handle_note TEXT,
    
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 4.5 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 飞书个人版不支持统计API | 功能无法实现 | 考虑升级企业版，或使用替代方案（如自建访问统计） |
| API调用频率限制 | 数据同步延迟 | 增量同步、批量查询、本地缓存 |
| 统计数据延迟 | 预警不及时 | 设置合理预期，增加其他检测维度 |
| Token过期处理 | 服务中断 | 自动刷新机制 + 提前预警 |

### 4.6 替代方案

如果飞书API无法满足需求，考虑以下替代方案：

1. **自建访问统计**
   - 在iframe加载时记录访问日志
   - 通过Referer判断内嵌/外部访问
   - 局限：只能统计本系统内的访问，无法检测直接访问飞书链接的情况

2. **第三方统计工具**
   - 集成百度统计/Google Analytics
   - 局限：同样只能统计本系统内访问

3. **飞书企业版升级**
   - 获取完整API能力
   - 局限：增加成本
