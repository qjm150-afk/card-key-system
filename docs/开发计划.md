## 五、功能详细规划

### 5.1 统计报表详细设计

#### 数据指标

| 指标名称 | 计算方式 | 展示形式 |
|---------|---------|---------|
| 总卡密数 | COUNT(*) | 数字卡片 |
| 有效卡密 | COUNT(status=1) | 数字卡片 |
| 已过期卡密 | COUNT(expire_time < now) | 数字卡片 |
| 即将过期(7天) | COUNT(expire_time < now+7d) | 数字卡片 + 列表 |
| 今日验证次数 | COUNT(access_logs today) | 数字卡片 |
| 本周新增卡密 | COUNT(created this week) | 趋势图 |
| 销售状态分布 | GROUP BY sales_status | 饼图 |
| 设备绑定分布 | GROUP BY device_count | 柱状图 |

#### API 接口

```
GET /api/admin/statistics/overview     # 总览数据
GET /api/admin/statistics/trend        # 趋势数据（按日/周/月）
GET /api/admin/statistics/distribution # 分布数据
```

### 5.4 链接健康检测详细设计

#### 检测逻辑

```
1. 定时任务：每天凌晨检测一次
2. 手动触发：后台按钮立即检测
3. 检测方式：HEAD请求飞书链接，检查HTTP状态码
4. 状态标记：
   - 正常 (200)
   - 失效 (404/403/500等)
   - 未知 (超时/网络错误)
```

#### 后台展示

```
┌─────────────────────────────────────┐
│ 🔗 链接健康状态                       │
├─────────────────────────────────────┤
│ 正常: 15 个                          │
│ 失效: 2 个   [查看详情]              │
│ 未知: 0 个                          │
│ [立即检测]                           │
└─────────────────────────────────────┘
```
