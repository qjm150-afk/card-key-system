## 四、本项目开发规则

### 4.1 核心原则

> **本项目设计目标：会员过期后仍能继续运行**

为实现此目标，本项目采用以下架构：
- ✅ 纯 FastAPI 后端 + 静态前端
- ✅ 不调用任何大语言模型
- ✅ 不使用图像生成等内置集成服务
- ✅ 不调用扣子工作流或智能体 API

### 4.2 本地/云端数据库隔离

本项目实现了本地开发与云端部署的数据库隔离：

| 环境 | 数据库 | 触发条件 |
|------|--------|----------|
| 本地开发 | SQLite (`/tmp/card_key_test.db`) | 无 `COZE_SUPABASE_URL` 环境变量 |
| 云端部署 | Coze Supabase | 有 `COZE_SUPABASE_URL` 环境变量 |

**实现文件：** `src/storage/database/db_client.py`

**工作原理：**
```python
from storage.database.db_client import get_db_client

# 自动根据环境选择数据库
client, is_sqlite = get_db_client()

# 使用方式与 Supabase 完全一致
response = client.table('card_keys_table').select('*').execute()
```

**优势：**
- 本地测试数据完全隔离，不会影响生产环境
- 无需额外配置，自动识别环境
- 零成本（不消耗 Coze 资源配额）

### 4.3 开发约束

#### 允许 ✅

- 常规数据库 CRUD 操作
- 纯 Python 业务逻辑计算
- 前端 HTML/CSS/JS 静态页面
- iframe 嵌入第三方内容（如飞书）
- 文件上传下载（不涉及 AI 处理）

#### 禁止 ❌

- 调用 `coze_coding_dev_sdk.llm` 等大模型接口
- 使用扣子工作流 API
- 集成图像生成、音视频处理服务
- 任何需要消耗积分的内置集成

### 4.4 代码规范

```python
# ✅ 允许：纯数据库操作
response = client.table('card_keys_table').select('*').execute()

# ✅ 允许：纯业务逻辑
def validate_card_key(key: str) -> bool:
    return len(key) == 19 and key.startswith('CSS-')

# ❌ 禁止：调用大语言模型
from coze_coding_dev_sdk.llm import LLMClient  # 不允许

# ❌ 禁止：调用工作流
response = client.workflows.run(...)  # 不允许
```

## 五、数据安全策略

### 5.1 部署前备份（强制）

**每次部署前必须执行数据备份！**

```bash
# 备份数据
python scripts/backup_data.py

# 查看备份列表
python scripts/restore_data.py --list
```

**备份存储：**
- 位置：`/backups/` 目录
- 保留：永久保留
- 格式：`backup_YYYYMMDD_HHMMSS.json`

### 5.2 数据恢复

**如果部署后出现问题，可以恢复数据：**

```bash
# 恢复最新备份
python scripts/restore_data.py --latest

# 恢复指定备份
python scripts/restore_data.py --file backup_20260313_120000.json
```

⚠️ **警告：恢复操作会清空目标表并重新导入数据，请谨慎使用！**

### 5.3 部署方式

#### 方式一：AI 自动部署（推荐）

直接告诉 AI："部署上线"或"准备部署"，AI 会自动执行：
1. 数据备份
2. 确认检查清单
3. 执行部署

#### 方式二：脚本自动部署

执行一条命令，自动完成备份 + 部署：

```bash
bash scripts/deploy.sh
```

流程：数据备份 → 确认 → 构建部署

#### 方式三：手动部署

```bash
# 1. 备份数据
python scripts/backup_data.py

# 2. 部署
coze build && coze start
```

### 5.4 部署检查清单

**每次部署前必须检查：**

- [ ] 1. 是否已执行数据备份？
- [ ] 2. 新功能是否涉及数据库结构变更？
- [ ] 3. 是否在本地测试环境验证通过？
- [ ] 4. 是否检查了代码中禁止的操作（LLM调用等）？

**部署后验证：**

- [ ] 1. 首页是否正常访问？
- [ ] 2. 卡密验证功能是否正常？
- [ ] 3. 管理后台是否正常登录？
- [ ] 4. 数据是否完整（卡密数量、日志数量）？

### 5.5 迁移方案（备用）

如需迁移到独立 Supabase 服务：

| 步骤 | 说明 |
|------|------|
| 1 | 注册 Supabase 官方账号 (supabase.com) |
| 2 | 创建新项目，获取 URL 和 API Key |
| 3 | 导出当前数据库数据 |
| 4 | 在新项目创建相同表结构 |
| 5 | 导入数据 |
| 6 | 修改环境变量配置 |
| 7 | 重新部署 |

**Supabase 官方免费版额度：**
- 数据库存储：500 MB
- 文件存储：1 GB
- 带宽：5 GB/月
- API 请求：无限制
