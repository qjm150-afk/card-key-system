# 卡密验证系统开发规范

## 一、会员过期影响与应对策略

### 1.1 会员过期后状态

| 项目 | 状态 |
|------|------|
| 套餐等级 | 回退到个人免费版 |
| 订阅积分 | ❌ 不可用 |
| 增购积分 | ✅ 仍可使用 |
| 活动积分 | ✅ 仍可使用 |
| 超额资源 | ✅ 保留存量 |

### 1.2 服务关停规则

| 积分状态 | 影响 |
|---------|------|
| 积分为 0 | 停止服务，拦截所有可能产生资源消耗的请求 |
| 持续 14 天为 0 | 记忆库被删除（不可恢复），其他资源暂不删除 |

### 1.3 个人免费版限制

| 限制项 | 免费版额度 |
|--------|-----------|
| 云端沙箱磁盘 | 10GB 上限 |
| 上传文件 | 有限制 |
| 多人协作 | 有限制 |

## 二、积分消耗规则

### 2.1 会消耗积分的操作 ⚠️

| 操作类型 | 说明 |
|----------|------|
| 开发对话 | 通过自然语言让扣子 AI 开发/修改项目 |
| 大语言模型调用 | LLM 推理、对话生成等 |
| 图像生成 | AI 图片生成服务 |
| API 请求 | 工作流调用、智能体调用等 |
| 音视频处理 | 音视频转码、处理等 |
| 知识库写入 | 知识库数据写入操作 |

### 2.2 不会消耗积分的操作 ✅

| 操作类型 | 说明 |
|----------|------|
| 项目托管/部署 | 部署后项目托管不消耗积分 |
| 静态资源访问 | 已部署的前端页面、静态文件 |
| 纯前端逻辑 | 本地计算、DOM 操作等 |
| 纯后端计算 | FastAPI 业务逻辑处理 |
| 常规数据库操作 | 单纯的数据存储和查询（"可能"不消耗） |

## 三、本项目开发规则

### 3.1 核心原则

> **本项目设计目标：会员过期后仍能继续运行**

为实现此目标，本项目采用以下架构：
- ✅ 纯 FastAPI 后端 + 静态前端
- ✅ 不调用任何大语言模型
- ✅ 不使用图像生成等内置集成服务
- ✅ 不调用扣子工作流或智能体 API

### 3.2 开发约束

#### 允许 ✅

- 常规数据库 CRUD 操作
- 纯 Python 业务逻辑计算
- 前端 HTML/CSS/JS 静态页面
- iframe 嵌入第三方内容（如飞书）
- 文件上传下载（不涉及 AI 处理）

#### 禁止 ❌

- 调用 `coze_coding_dev_sdk.llm` 等大模型接口
- 使用扣子工作流 API
- 集成图像生成、音视频处理服务
- 任何需要消耗积分的内置集成

### 3.3 代码规范

```python
# ✅ 允许：纯数据库操作
response = client.table('card_keys_table').select('*').execute()

# ✅ 允许：纯业务逻辑
def validate_card_key(key: str) -> bool:
    return len(key) == 19 and key.startswith('CSS-')

# ❌ 禁止：调用大语言模型
from coze_coding_dev_sdk.llm import LLMClient  # 不允许

# ❌ 禁止：调用工作流
response = client.workflows.run(...)  # 不允许
```

## 四、数据安全策略

### 4.1 定期备份

建议每周或每月备份一次数据库数据。

### 4.2 迁移方案（备用）

如需迁移到独立 Supabase 服务：

| 步骤 | 说明 |
|------|------|
| 1 | 注册 Supabase 官方账号 (supabase.com) |
| 2 | 创建新项目，获取 URL 和 API Key |
| 3 | 导出当前数据库数据 |
| 4 | 在新项目创建相同表结构 |
| 5 | 导入数据 |
| 6 | 修改环境变量配置 |
| 7 | 重新部署 |

**Supabase 官方免费版额度：**
- 数据库存储：500 MB
- 文件存储：1 GB
- 带宽：5 GB/月
- API 请求：无限制

## 六、风险预警

### 6.1 当前风险项
