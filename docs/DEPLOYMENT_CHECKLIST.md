# 部署前检查清单 (Pre-deployment Checklist)

> ⚠️ **重要**：每次部署前必须按顺序完成以下所有检查项，确保系统安全稳定上线。

---

## 📋 检查清单

### 第一阶段：准备工作

| 序号 | 检查项 | 操作 | 状态 |
|------|--------|------|------|
| 1.1 | 阅读项目摘要 | 检查「关键决策」中是否有特殊要求 | ☐ |
| 1.2 | 检查注意事项 | 确认是否有风险点需要规避 | ☐ |
| 1.3 | 确认代码变更 | 了解本次部署包含哪些修改 | ☐ |

### 第二阶段：数据安全 ⭐

| 序号 | 检查项 | 操作命令 | 状态 |
|------|--------|----------|------|
| 2.1 | **数据备份** | `python scripts/backup_data.py` | ☐ |
| 2.2 | 确认备份成功 | 检查 `backups/` 目录生成新文件 | ☐ |
| 2.3 | 记录备份文件名 | `backup_YYYYMMDD_HHMMSS.json` | ☐ |

### 第三阶段：构建验证

| 序号 | 检查项 | 操作命令 | 状态 |
|------|--------|----------|------|
| 3.1 | 执行构建 | `coze build` | ☐ |
| 3.2 | 检查构建日志 | 确认无 ERROR 级别日志 | ☐ |
| 3.3 | 确认依赖安装 | 所有 requirements 已安装 | ☐ |

### 第四阶段：服务验证

| 序号 | 检查项 | 操作命令 | 状态 |
|------|--------|----------|------|
| 4.1 | 检查服务状态 | `curl -I http://localhost:5000` | ☐ |
| 4.2 | 测试用户登录页 | 访问 `http://localhost:5000/` | ☐ |
| 4.3 | 测试管理后台 | 访问 `http://localhost:5000/admin` | ☐ |
| 4.4 | 测试核心 API | 卡密验证、登录、数据查询 | ☐ |

### 第五阶段：日志检查

| 序号 | 检查项 | 操作命令 | 状态 |
|------|--------|----------|------|
| 5.1 | 检查应用日志 | `tail -n 50 /app/work/logs/bypass/app.log` | ☐ |
| 5.2 | 确认无严重错误 | 无 ERROR/Exception/Traceback | ☐ |

### 第六阶段：完成部署

| 序号 | 检查项 | 状态 |
|------|--------|------|
| 6.1 | 所有检查项通过 | ☐ |
| 6.2 | 记录部署时间 | ☐ |
| 6.3 | 通知相关人员 | ☐ |

---

## 🔧 快速部署命令

### 方式一：使用部署脚本（推荐）
```bash
./scripts/deploy.sh
```

### 方式二：手动执行
```bash
# 1. 数据备份
python scripts/backup_data.py

# 2. 构建
coze build

# 3. 验证服务
curl -I http://localhost:5000
```

## ⚠️ 回滚指南

如果部署后发现问题，可执行数据恢复：

```bash
# 查看备份列表
ls -la backups/

# 恢复数据
python scripts/restore_data.py --file backups/backup_YYYYMMDD_HHMMSS.json
```
