# 合规风险检查文档

> 本文档基于《中华人民共和国个人信息保护法》《中华人民共和国网络安全法》《中华人民共和国数据安全法》等法律法规制定，用于指导项目开发过程中的合规实践。

---

## 一、法律框架概述

### 1.1 核心法律法规

| 法律法规 | 施行日期 | 核心要求 |
|---------|---------|---------|
| 《个人信息保护法》 | 2021年11月1日 | 个人信息处理需遵循合法、正当、必要原则，需取得个人同意 |
| 《网络安全法》 | 2017年6月1日 | 网络运营者应采取技术措施防止个人信息泄露、毁损、丢失 |
| 《数据安全法》 | 2021年9月1日 | 开展数据处理活动应依法建立数据安全管理制度 |
| 《网络数据安全管理条例》 | 2025年1月1日 | 细化数据处理者义务，明确个人信息处理规则 |

### 1.2 重要国家标准

| 标准编号 | 标准名称 | 适用场景 |
|---------|---------|---------|
| GB/T 35273-2020 | 个人信息安全规范 | 个人信息处理全流程 |
| GB/T 37988-2019 | 数据安全能力成熟度模型 | 数据安全能力建设 |
| GB/T 41479-2022 | 网络数据处理安全要求 | 网络数据处理活动 |

## 二、个人信息识别清单

### 2.1 个人信息定义

根据《个人信息保护法》，个人信息是指"以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息"。

### 2.2 常见个人信息类型

| 类型 | 示例 | 风险等级 | 本项目是否涉及 |
|------|------|---------|---------------|
| **身份信息** | 姓名、身份证号、护照号 | 🔴 高 | ❌ 否 |
| **联系方式** | 手机号、邮箱、地址 | 🔴 高 | ❌ 否 |
| **网络标识** | IP地址、MAC地址、设备ID | 🔴 高 | ❌ **已移除** |
| **设备指纹** | User-Agent、设备类型 | 🔴 高 | ❌ **已移除** |
| **生物特征** | 人脸、指纹、声纹 | 🔴 高 | ❌ 否 |
| **金融信息** | 银行卡号、支付账号 | 🔴 高 | ❌ 否 |
| **位置信息** | GPS坐标、地址 | 🔴 高 | ❌ 否 |
| **行为信息** | 浏览记录、访问日志 | 🟡 中 | ⚠️ 仅保留访问时间 |
| **账号信息** | 用户名、密码 | 🔴 高 | ❌ 否（仅管理后台密码） |

> **合规更新 (2026-03-27)**：
> - API 限流功能使用 `card_key`（卡密前8位）作为标识，**不收集 IP**
> - 登录安全功能使用全局计数器，**不收集 IP**
> - 日志中不记录任何 IP 地址

### 2.3 敏感个人信息

根据《个人信息保护法》第二十八条，敏感个人信息包括：

| 类型 | 示例 | 处理要求 |
|------|------|---------|
| 生物识别信息 | 人脸、指纹 | 需单独同意 + 明确必要性 |
| 宗教信仰信息 | 宗教信仰 | 需单独同意 |
| 特定身份信息 | 残疾证、军官证 | 需单独同意 |
| 医疗健康信息 | 病历、诊断 | 需单独同意 |
| 金融账户信息 | 银行卡、证券账户 | 需单独同意 |
| 行踪轨迹信息 | 位置、轨迹 | 需单独同意 |
| 不满14周岁信息 | 儿童个人信息 | 需监护人同意 |

**本项目承诺：不收集任何敏感个人信息。**

## 三、本项目数据合规清单

### 3.1 收集的数据项

| 数据项 | 类型 | 用途 | 合规性 | 备注 |
|-------|------|------|-------|------|
| 卡密值 | 业务数据 | 产品验证 | ✅ 合规 | 非个人信息 |
| 卡密状态 | 业务数据 | 业务管理 | ✅ 合规 | 非个人信息 |
| 飞书URL | 业务数据 | 内容展示 | ✅ 合规 | 非个人信息 |
| 飞书密码 | 业务数据 | 内容访问 | ✅ 合规 | 非个人信息 |
| 有效期 | 业务数据 | 业务管理 | ✅ 合规 | 非个人信息 |
| 销售渠道 | 业务数据 | 渠道分析 | ✅ 合规 | 非个人信息 |
| 订单号 | 业务数据 | 订单关联 | ✅ 合规 | 非个人信息 |
| 访问时间 | 行为数据 | 安全审计 | ✅ 合规 | 最小必要 |
| 访问结果 | 业务数据 | 业务统计 | ✅ 合规 | 非个人信息 |
| 设备标识 | 业务数据 | 设备限制 | ✅ 合规 | 本地存储，非个人信息 |

### 3.2 已移除的高风险数据

| 数据项 | 原用途 | 移除原因 | 移除日期 |
|-------|--------|---------|---------|
| **IP地址** | 安全审计、地域分析 | 属于个人信息，合规风险高 | 2026-03-13 |
| **User-Agent** | 设备类型分析 | 包含设备指纹信息，属于个人信息 | 2026-03-13 |
| **设备类型** | 设备分布统计 | 来源于User-Agent，属于个人信息 | 2026-03-13 |

## 四、开发合规检查清单

### 4.1 数据收集检查

- [ ] 确认收集的数据项是否属于个人信息
- [ ] 如属于个人信息，是否遵循"最小必要"原则
- [ ] 是否已向用户告知数据收集目的
- [ ] 是否已获得用户同意（如需要）
- [ ] 是否有隐私政策说明

### 4.2 数据存储检查

- [ ] 敏感数据是否加密存储
- [ ] 数据库访问是否有权限控制
- [ ] 是否有数据备份机制
- [ ] 是否有数据保留期限设置

### 4.3 数据使用检查

- [ ] 数据使用是否超出收集时声明的目的
- [ ] 是否将数据提供给第三方
- [ ] 是否有数据导出/共享记录

### 4.4 数据删除检查

- [ ] 是否提供用户数据删除途径
- [ ] 过期数据是否自动清理
- [ ] 数据删除是否彻底

## 五、高风险行为禁止清单

### 5.1 绝对禁止

| 禁止行为 | 法律依据 | 违法后果 |
|---------|---------|---------|
| ❌ 收集与业务无关的个人信息 | 《个保法》第6条 | 责令改正、警告、没收违法所得、罚款 |
| ❌ 未经同意收集个人信息 | 《个保法》第13条 | 同上 |
| ❌ 收集敏感个人信息未单独同意 | 《个保法》第29条 | 同上 |
| ❌ 泄露、篡改、丢失个人信息 | 《个保法》第51条 | 民事责任、行政处罚、刑事责任 |
| ❌ 非法买卖个人信息 | 《刑法》第253条 | 刑事责任 |
| ❌ 向境外提供个人信息未安全评估 | 《个保法》第38条 | 行政处罚 |

### 5.2 本项目特别禁止

| 禁止行为 | 替代方案 |
|---------|---------|
| ❌ 收集用户IP地址 | 不收集（已移除） |
| ❌ 收集用户User-Agent | 不收集（已移除） |
| ❌ 收集设备类型/设备指纹 | 使用本地Session标识 |
| ❌ 追踪用户行为轨迹 | 仅记录验证成功/失败 |
| ❌ 分析用户地理位置 | 不进行地域分析（已移除） |

## 七、合规自查流程

### 7.1 新功能上线前检查

```
1. 是否涉及新的数据收集？
   ↓ 是
2. 该数据是否属于个人信息？
   ↓ 是
3. 是否遵循最小必要原则？
   ↓ 是
4. 是否已在隐私政策中说明？
   ↓ 是
5. 是否需要用户同意？
   ↓ 是
6. 实施同意机制
   ↓
7. 上线
```

### 7.2 定期自查（建议每季度）

- [ ] 检查数据收集范围是否扩大
- [ ] 检查数据存储是否安全
- [ ] 检查是否有未授权的数据访问
- [ ] 检查日志是否包含敏感信息
- [ ] 检查隐私政策是否更新

## 八、应急响应

### 8.1 数据泄露应急预案

| 阶段 | 行动 | 责任人 |
|------|------|--------|
| 发现 | 立即停止相关服务，保护现场 | 运维人员 |
| 报告 | 72小时内向监管部门报告 | 项目负责人 |
| 通知 | 及时通知受影响用户 | 项目负责人 |
| 整改 | 查明原因，修补漏洞 | 开发团队 |
| 总结 | 形成事件报告，完善制度 | 项目负责人 |
