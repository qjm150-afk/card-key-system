## 迁移脚本详情

### 001_add_analytics_fields.sql

**变更日期**：2026-03-12

**变更内容**：
- 为 `card_keys_table` 添加统计分析相关字段
- 添加 `link_click_count` 链接点击次数字段
- 添加 `last_access_time` 最后访问时间字段

**SQL 内容**：
```sql
-- 添加链接点击次数字段
ALTER TABLE card_keys_table ADD COLUMN link_click_count INTEGER DEFAULT 0;

-- 添加最后访问时间字段
ALTER TABLE card_keys_table ADD COLUMN last_access_time TIMESTAMP;

-- 创建索引以优化查询性能
CREATE INDEX IF NOT EXISTS idx_card_keys_last_access ON card_keys_table(last_access_time);
```

## 执行方法

### 本地 SQLite 环境

```bash
# 执行单个迁移脚本
sqlite3 /tmp/card_key_test.db < src/migrations/001_add_analytics_fields.sql

# 查看迁移结果
sqlite3 /tmp/card_key_test.db ".schema card_keys_table"
```

### Supabase 云端环境

1. 登录 Supabase 控制台
2. 进入 SQL Editor
3. 粘贴迁移脚本内容
4. 点击 Run 执行

**注意**：Supabase 执行 ALTER TABLE 时如果字段已存在会报错，需要先检查字段是否存在：

```sql
-- 检查字段是否存在（PostgreSQL）
SELECT column_name 
FROM information_schema.columns 
WHERE table_name = 'card_keys_table' 
AND column_name = 'session_id';
```

## 回滚指南

### SQLite 回滚

SQLite 不支持 DROP COLUMN，如需回滚需要：

1. 导出数据
2. 创建新表（不含目标字段）
3. 导入数据
4. 重命名表

```bash
# 备份当前数据
python scripts/backup_data.py

# 手动重建表结构
# 具体操作请参考 SQLite 官方文档
```

### Supabase 回滚

PostgreSQL 支持 DROP COLUMN：

```sql
-- 删除字段
ALTER TABLE card_keys_table DROP COLUMN link_name;

-- 删除索引
DROP INDEX IF EXISTS idx_card_keys_link_name;
```

## 注意事项

1. **执行前备份**：每次迁移前执行 `python scripts/backup_data.py`
2. **测试环境验证**：先在本地 SQLite 测试迁移脚本
3. **字段检查**：Supabase 执行前检查字段是否已存在
4. **索引冲突**：创建索引前检查是否已存在同名索引
