# 压缩摘要

## 用户需求与目标
- 原始目标: 创建卡密验证系统，嵌入飞书多维表格内容，隐藏"进入原应用"按钮
- 当前目标: ✅ 已完成"已过期"统计卡片同步到卡种卡密管理模态框
- 当前状态: ✅ 开发完成，功能完整，运行稳定

## 项目概览
- 概述: 卡密验证系统，用户输入卡密后访问飞书内容，包含管理后台和前端用户页面
- 技术栈: FastAPI, PostgreSQL(Supabase), HTML/CSS/JavaScript, coze-coding-dev-sdk
- 编码规范: 遵循《个人信息保护法》，不收集IP、UA等个人信息；修改后需检查对其他地方的影响并同步修改

## 问题或错误及解决方案
- 问题: 卡种列表拖拽排序后刷新页面，顺序恢复原状
  - 根本原因: Supabase Python 客户端的链式 `.order()` 调用存在顺序问题，第二个 order 会覆盖第一个
  - 解决方案: 简化排序逻辑，只使用单字段排序 `.order('sort_order')`
- 问题: 后台报错 `column "sort_order" does not exist`
  - 根本原因: 卡密列表 API 错误使用了 `sort_order` 排序，但卡密表没有该字段
  - 解决方案: 卡密列表改用 `.order('id', desc=True)` 排序
- 问题: 卡种管理页面首次加载不显示数据
  - 根本原因: 默认显示卡种管理页面，但登录成功后 `showMainContent()` 没有调用 `loadCardTypes()`
  - 解决方案: 在 `showMainContent()` 中添加 `loadCardTypes()` 调用
- 问题: 切换到卡密管理页面后数据不刷新
  - 根本原因: `switchTab()` 函数遗漏了 `'cards'` 标签页的处理
  - 解决方案: 添加 `if (tab === 'cards')` 分支，调用 `loadCards()` 和 `loadStats()`
- 问题: 操作日志页面缺少"导入卡密"操作类型
  - 解决方案: 在操作类型筛选下拉框和表格渲染映射中添加 `import_cards` 类型
- 问题: 访问日志销售状态筛选分页不准确
  - 解决方案: 改为在数据库层面筛选，先查询符合销售状态的卡密列表，再使用 `in_('key_value', key_values)` 筛选
- 问题: 统计卡片数据不准确（只取前100条）
  - 解决方案: 新增专用统计API `/api/admin/cards/stats`，后端直接查询数据库计算
- 问题: 管理后台点击搜索/重置按钮后右上角出现空白粉色椭圆框
  - 解决方案: 
    1. `loadCards` 函数支持解析 FastAPI 的 `detail` 格式错误消息（`result.msg || result.detail || '加载失败'`）
    2. `showToast` 函数处理空消息（`message || (type === 'error' ? '操作失败' : '操作成功')`）
- 问题: 生产环境部署后管理后台数据为空
  - 根本原因: 
    1. Coze 平台采用**开发环境与生产环境完全隔离**的设计
    2. **首次部署时未勾选"同步开发环境的数据至生产环境"选项**
    3. **后续部署仅同步表结构，不会同步具体数据**
    4. 用户在生产环境添加的数据，在某次重新部署后被重置丢失
  - 解决方案: 
    1. **推荐**：在 Coze 平台部署页面，勾选"同步开发环境的数据至生产环境"后重新部署
    2. **备选**：使用 `scripts/import_data_to_production.py` 手动导入数据
  - 数据迁移工具:
    - 导出文件: `data_export_for_production.json`（开发环境数据快照）
    - 导入脚本: `scripts/import_data_to_production.py`（在生产环境运行）
