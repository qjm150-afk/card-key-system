## 二、关键决策

### 2.1 CSS裁剪隐藏"进入原应用"按钮

**问题**：飞书多维表格嵌入时，顶部显示"进入原应用"按钮，存在导流风险。

**方案尝试**：
- ❌ 方案1：URL参数 `?hideTitleBar=true` - 飞书侧不支持
- ✅ 方案2：CSS裁剪 `clip-path` - 完美解决

**最终实现**：
```css
.feishu-iframe {
  clip-path: inset(0 0 0 0);
  margin-top: -50px;
  height: calc(100% + 50px);
}
```

**最佳裁剪高度**：50px

### 2.2 博主在线状态模拟

**问题**：如何展示博主的在线状态？

**方案**：前端模拟，基于时段概率分布
- 工作时段 (9:00-18:00)：高概率在线
- 晚间时段 (18:00-23:00)：中等概率
- 深夜时段 (23:00-9:00)：低概率

**优势**：
- 不依赖后端状态
- 无需维护在线状态同步
- 用户体验自然

### 2.3 退出按钮融合设计

**设计**：退出按钮与在线人数图标融合
- 默认显示：博主状态 + 在线人数
- 点击展开：显示退出登录按钮

**优势**：
- UI更简洁
- 操作逻辑清晰

## 四、问题解决记录

### 4.1 统计卡片数据逻辑

**问题**：统计卡片"有效卡密"和"已激活"定义不清

**解决方案**：
- **已售出**：销售状态为 `sold` 的卡密
- **已激活**：有设备绑定的卡密

### 4.2 Supabase 客户端不支持 `not_.in_()`

**问题**：激活状态筛选在 Supabase 客户端不支持 `not_.in_()` 方法

**解决方案**：使用应用层过滤处理 NULL 值问题

### 4.3 SQLite 客户端缺少逻辑运算方法

**问题**：SQLite 客户端缺少 `or_()` 和 `not_()` 方法

**解决方案**：
1. 创建 `SQLiteNotWrapper` 类
2. 扩展 `SQLiteClient` 添加 `or_()` 和 `not_()` 方法
3. 保持与 Supabase 客户端 API 一致
