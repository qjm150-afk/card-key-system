## 二、关键决策

### 2.1 CSS裁剪隐藏"进入原应用"按钮

**问题**：飞书多维表格嵌入时，顶部显示"进入原应用"按钮，存在导流风险。

**方案尝试**：
- ❌ 方案1：URL参数 `?hideTitleBar=true` - 飞书侧不支持
- ✅ 方案2：CSS裁剪 `clip-path` - 完美解决

**最终实现**：
```css
.feishu-iframe {
  clip-path: inset(0 0 0 0);
  margin-top: -50px;
  height: calc(100% + 50px);
}
```

**最佳裁剪高度**：50px

### 2.2 博主在线状态模拟

**问题**：如何展示博主的在线状态？

**方案**：前端模拟，基于时段概率分布
- 工作时段 (9:00-18:00)：高概率在线
- 晚间时段 (18:00-23:00)：中等概率
- 深夜时段 (23:00-9:00)：低概率

**优势**：
- 不依赖后端状态
- 无需维护在线状态同步
- 用户体验自然

### 2.3 退出按钮融合设计

**设计**：退出按钮与在线人数图标融合
- 默认显示：博主状态 + 在线人数
- 点击展开：显示退出登录按钮

**优势**：
- UI更简洁
- 操作逻辑清晰

## 四、问题解决记录

### 4.1 统计卡片数据逻辑

**问题**：统计卡片"有效卡密"和"已激活"定义不清

**解决方案**：
- **已售出**：销售状态为 `sold` 的卡密
- **已激活**：有设备绑定的卡密

### 4.2 Supabase 客户端不支持 `not_.in_()`

**问题**：激活状态筛选在 Supabase 客户端不支持 `not_.in_()` 方法

**解决方案**：使用应用层过滤处理 NULL 值问题

### 4.3 SQLite 客户端缺少逻辑运算方法

**问题**：SQLite 客户端缺少 `or_()` 和 `not_()` 方法

**解决方案**：
1. 创建 `SQLiteNotWrapper` 类
2. 扩展 `SQLiteClient` 添加 `or_()` 和 `not_()` 方法
3. 保持与 Supabase 客户端 API 一致

### 4.4 筛选下拉框选项数量动态更新（2026-03-15）

**问题**：点击筛选下拉选项后，所有下拉选项只显示1个选项

**根本原因**：
1. 服务启动时未加载 `.env.local` 环境变量，导致连接到云端 Supabase 而非本地 SQLite
2. 后端 API 为所有字段应用了相同的筛选条件，导致每个字段的选项都只反映当前筛选结果

**解决方案**：
1. 在 `src/main.py` 开头添加加载 `.env.local` 环境变量的代码
2. 重构 `/api/admin/cards/filter-options` API，为每个字段分别执行查询：
   - 每个字段排除自身的筛选条件
   - 其他字段保留当前筛选条件

**核心代码变更**：
```python
# 加载环境变量
from dotenv import load_dotenv
_env_local = os.path.join(_parent_dir, '.env.local')
if os.path.exists(_env_local):
    load_dotenv(_env_local)

# 为每个字段分别构建查询
def build_query(exclude: str = None):
    # 排除指定字段的筛选条件
    if status and exclude != 'status':
        query = query.eq('status', status)
    # ...
```

### 4.5 筛选下拉框选中值不显示（2026-03-15）

**问题**：用户选中销售渠道、过期时间、链接名称筛选框中的选项后，列表正确显示筛选结果，但选中的选项不会在筛选框上显示

**根本原因**：`updateSelectDropdown` 函数使用 CSS 选择器 `option[value="${currentValue}"]` 来查找选项并恢复选中值，但如果值包含特殊字符（如中文"淘宝"），CSS 选择器可能无法正确匹配

**解决方案**：改用遍历选项的方式恢复选中值

**核心代码变更**：
```javascript
// 恢复选中值（遍历选项查找匹配，避免 CSS 选择器特殊字符问题）
if (currentValue) {
    const options = select.options;
    for (let i = 0; i < options.length; i++) {
        if (options[i].value === currentValue) {
            select.value = currentValue;
            break;
        }
    }
}
```
