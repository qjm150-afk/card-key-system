## 一、问题描述

### 现象
在 iframe 中嵌入飞书多维表格应用模式时，页面底部出现「进入原应用」按钮，用户点击后可跳转到飞书原生应用，影响卡密验证系统的封闭性。

### 官方声明
> 目前 iframe 形式嵌套多维表格支持不完善，可能会出现不兼容的 bug，后续可能会有调整。请知悉可能出现的问题，谨慎使用。
> 
> —— 来源：飞书内部技术文档

### 根因分析
根据飞书产品逻辑，按钮出现与以下因素相关：
1. **飞书版本更新**：2025年底至2026年初全量开放应用模式后新增此引导按钮
2. **权限配置变更**：分享范围、角色权限变化可能触发按钮显示
3. **安全策略限制**：域名未加白名单、iframe sandbox 属性限制等
4. **用户身份变化**：未登录或身份切换

---

## 二、当前解决方案

### 方案1：官方嵌入参数（已实现，无效）
```python
# src/main.py - add_feishu_embed_params()
参数列表：
- hideHeader=1    # 隐藏头部
- hideSidebar=1   # 隐藏侧边栏  
- vc=true         # 隐藏新增视图，工具栏上移
```

**状态**：❌ 已添加但无法隐藏「进入原应用」按钮

### 方案2：CSS 遮罩层（当前使用）
```css
/* src/static/index.html */

/* 顶部右侧遮罩 */
.feishu-top-right-mask {
    width: 220px;
    height: 80px;
    z-index: 9999;
}

/* 底部遮罩 - 覆盖"进入原应用"按钮 */
.feishu-bottom-mask {
    height: 60px;
    z-index: 9999;
    pointer-events: auto;
}
```

**状态**：✅ 有效，但属于临时方案
